试着实现一个Vue

Posted on 2018-04-08

接触Vue快一年了，也大致了解其中的相关原理，下面参考网上的一些资料，试着实现一个自己的VUE

首先是视图，因为在大型应用中，为了控制数据的逻辑操作，数据与展现必须分离，不像JQuery，数据与展现牢牢绑定在一起，当你更新某一部分视图时，必须把数据和视图都重新更新一遍，但其实在大多数应用里，对数据的操作是最多的。
按照MV*框架的设计理念，视图是其中的View层，源代码是需要进行编译的模板，编译后是真正浏览器解析的代码，假设有如下模板：


那么在script中我们是这样定义的：
var message = ‘I am Ovan’

如何把模板渲染出来呢，很简单，一个正则就匹配出来了，但是这样只能渲染这一个变量，假设一个页面用了很多数据，那就要建很多message这种变量，一方面渲染效率就降低了，不能像直接设置innerHTML那样直接更新整个页面，另一方面这么多变量维护起来也不容易。我们可以把一个页面或一个component需要的数据放在一个对象里面，也就是：
var data = {
message: ‘I am Ovan’
}
渲染的时候只渲染data.message

但是怎么做到当data.message改变的时候，div里面的message也跟着改变呢（怎么实现MVVM）？
三种思路：
①发布者/订阅者模式：
当设置data.message的值时 -> 发布了一个change事件，文档上进行监听，所有绑定data.message的元素进行数据变化的回调，即调用数据变化后的callback，类源代码如下：

②脏值检查，监测变量是否发生变化，angular1.0的思路是当指定事件发生时进行检查，如DOM变化事件，XHR响应事件，Timer事件

③数据劫持结合发布订阅者模式，劫持每个数据属性的setter,getter，当数据变动时，发布信息给订阅者，触发响应的回调

vue利用浏览器拥有的Object.defineProperty()劫持属性。所以要模拟vue，需要实现以下几个方面的功能：
1.实现一个数据observer, 对数据对象的所有属性进行监听，有变动则发布消息给订阅者
2.编译模板，对每个元素绑定的数据进行解析
3.实现一个watcher，当收到通知时，执行指令绑定的响应回调函数
